ARCHITECTURE

4계층 클린 아키텍처 · 단방향 의존

UIlib/ui — 화면 · 위젯, 상태만 참조
↓ depends on
Statelib/state — Riverpod: GazeNotifier · InterviewNotifier · ReportNotifier
↓ depends on
Domainlib/domain — 순수 Dart 엔티티, 외부 import 없음
↑ implements
Datalib/data — local(Hive) · remote(OpenAI · ML Kit · Firebase)
의존 방향: ui → state → domain ← data · 원본 영상 미저장
lib/
├── ui/      # 화면·위젯
├── state/  # Riverpod Notifier
├── domain/ # 순수 Dart 엔티티
└── data/
    ├── local/  # Hive
    └── remote/ # OpenAI · ML Kit
ADR · 기술 의사결정

왜 이 기술을 선택했나

ADR결정기각 대안 · 이유
001FlutterReact Native · 네이티브 분리 — 단일 코드베이스로 iOS/Android 동시 대응
002RiverpodProvider · BLoC · GetX — 테스트 용이성과 타입 안전성
003OpenAI gpt-4o-miniGemini · 온디바이스 LLM — 한국어 품질과 JSON 응답 안정성
004ML KitMediaPipe · 자체 TFLite — 온디바이스 처리로 영상 미저장, 빠른 통합
005HiveSQLite · Isar — 가벼운 로컬 캐시 용도에 충분, 빌드 의존성 단순
각 ADR: Context · Decision · Status · Consequence + 기각 이유 (docs/ADR/)

TROUBLESHOOTING

응시율이 항상 0%였던 이유

01
문제
앱은 정상 실행되는데 응시율이 항상 0%, 에러 로그 없이 시선 분석만 조용히 실패
02
원인
ML Kit은 NV21만 받는데 카메라가 YUV_420_888로 프레임 전달
03
해결
camera_frame_converter.dart에서 row stride · pixel stride까지 처리해 NV21로 변환
04
검증
포맷 정합 후 FaceDetector가 시선을 정상 인식하도록 복구
