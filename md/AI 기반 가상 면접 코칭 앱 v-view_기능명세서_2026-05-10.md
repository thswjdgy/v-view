## 1. 프로젝트 개요
---
### 카테고리
> 생산성 / 업무

### 한 줄 정의
AI 기반 가상 면접 코칭 앱 v-view

### 배경
많은 면접 준비생들은 실제와 같은 면접 환경을 조성하기 어렵고, 혼자서 연습할 때 자신의 비언어적 표현이나 답변 내용을 객관적으로 평가하기 어렵다는 문제점을 겪고 있습니다. 기존의 온라인 자료나 강의만으로는 개인에게 특화된 맞춤형 피드백을 받기 어려워 실질적인 개선에 한계가 있습니다.

### 사용자 문제
사용자들은 혼자 면접을 준비할 때 실제와 같은 긴장감 있는 환경을 조성하기 어렵고, 자신의 비언어적 요소(시선, 표정, 제스처 등)나 언어적 표현(말 속도, 필러워드, 답변 내용)에 대해 객관적인 피드백을 받기 어렵습니다. 이로 인해 자신의 약점을 정확히 파악하고 개선하는 데 어려움을 겪습니다.

### 해결 방식
v-view는 AI 기반 카메라 및 음성 분석 기술을 활용하여 사용자의 면접 연습을 실시간으로 분석하고, 맞춤형 예상 질문 생성 및 종합 피드백 리포트를 제공함으로써 실제 면접과 유사한 환경을 구현하고 객관적인 개선 포인트를 제시합니다. 이를 통해 사용자는 자신의 강점과 약점을 명확히 인지하고 효과적으로 면접 역량을 강화할 수 있습니다.

### 차별점
경쟁사와 차별화되는 v-view의 핵심 가치는 '실시간 비언어 피드백'에 있습니다. 단순히 답변 내용을 평가하는 것을 넘어, AI가 실시간으로 사용자의 시선 처리, 표정, 제스처 등 비언어적 요소를 정밀하게 분석하여 즉각적인 개선 포인트를 제공합니다. 또한, 직종/회사/자기소개서 기반의 'AI 맞춤형 질문 생성' 및 상세한 '종합 피드백 리포트'를 통해 사용자가 면접 역량을 체계적으로 향상시킬 수 있도록 돕습니다.

### 타겟 사용자
주요 타겟 사용자는 취업 준비생, 이직 준비생, 대학 입시 준비생, 승진 대상자 등 면접을 앞두고 있는 모든 개인입니다. 특히, 면접 경험이 적거나 혼자 면접을 준비하며 객관적인 피드백에 대한 갈증을 느끼는 사용자, 반복적인 연습을 통해 약점을 개선하고 싶은 사용자에게 유용합니다.

### 사용 시나리오
사용자는 실제 면접 전, v-view 앱을 통해 최종 모의 면접을 진행합니다. 앱이 제공하는 직종/회사/자기소개서 기반 맞춤형 질문과 꼬리 질문에 답변하며 실전과 같은 긴장감을 느끼고 연습합니다. 면접이 진행되는 동안 AI는 사용자의 시선 처리, 말 속도 등을 실시간으로 분석합니다. 세션 종료 후에는 종합 피드백 리포트를 확인하여 자신의 비언어적 요소와 답변 내용에 대한 객관적인 평가를 받고, 개선 우선순위를 파악합니다. 사용자는 이 피드백을 바탕으로 반복적인 연습을 통해 약점을 개선하고 다음 면접에 대비합니다.

### 제품 목표
취업 및 대학 면접을 준비하는 사용자가 혼자서도 실전과 같은 환경에서 면접을 연습하고, AI의 객관적인 실시간 분석 및 맞춤형 피드백을 통해 자신의 강점과 약점을 명확히 파악하여 면접 역량을 효과적으로 개선하도록 지원합니다.

### 핵심 KPI
월간 활성 사용자(MAU) 증가율, 평균 세션 완료율 80% 이상, 피드백 리포트 조회율 90% 이상, 사용자당 평균 주간 면접 세션 수 3회 이상, 앱 스토어 평점 4.5점 이상, 사용자 만족도 설문조사 평균 4점 이상, 이전 세션 대비 리포트 지표 개선율.

### 리스크/이슈
AI 분석의 정확성 및 신뢰도 부족, 사용자 개인정보 보호 및 데이터 보안 문제, 경쟁사 등장 및 유사 서비스 출시, AI 학습 데이터 부족으로 인한 질문 생성 및 피드백 품질 저하, 네트워크 환경 불안정 시 서비스 지연 또는 오류 발생.

## 1. AI 기반 맞춤형 모의 면접 환경 제공
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 선택한 면접 유형(직무, 인성, 대학입시)과 입력한 정보(직종, 회사, 자기소개서)를 기반으로 AI가 맞춤 예상 질문과 꼬리 질문을 자동으로 생성하고, 타이머와 연속 질문 기능을 통해 실제 면접과 유사한 환경을 제공해야 합니다.

**수용 기준:**
```
1. ○ 사용자는 직무, 인성, 대학입시 면접 유형 중 하나를 선택할 수 있어야 한다.
2. ○ 사용자가 입력한 직종, 회사 정보, 자기소개서를 바탕으로 AI가 관련된 예상 질문을 3개 이상 생성해야 한다.
3. ○ 생성된 질문에 대한 답변에 기반하여 AI가 꼬리 질문을 1개 이상 자동으로 생성해야 한다.
4. ○ 모의 면접 진행 시 질문당 답변 시간 제한(타이머) 기능이 작동해야 한다.
5. ○ 모의 면접 진행 시 질문이 자동으로 다음 질문으로 전환되어야 한다.
```

## 2. 실시간 시선 처리 분석 및 피드백
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

모의 면접 진행 중 카메라를 통해 사용자의 시선 처리를 실시간으로 분석하고, 화면 응시 여부 및 시선 분산 정도를 감지하여 세션 종료 후 피드백 리포트에 포함해야 합니다.

**수용 기준:**
```
1. ○ 모의 면접 진행 중 사용자의 시선이 화면 중앙을 응시하는지 여부를 실시간으로 감지해야 한다.
2. ○ 사용자의 시선이 화면 외부로 분산되는 경우를 감지해야 한다.
3. ○ 세션 종료 후 생성되는 리포트에 시선 처리 분석 결과(예: 화면 응시율, 시선 분산 횟수 및 지속 시간)를 시각적으로 제공해야 한다.
4. ○ 시선 처리 분석 결과에 대한 개선 가이드라인을 제시해야 한다.
```

## 3. 종합 피드백 리포트 제공
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

각 면접 세션 종료 후, 시선 처리 분석 결과와 AI 질문에 대한 답변 내용을 종합하여 상세 피드백 리포트를 제공합니다. 리포트에는 개선 포인트 우선순위가 제시되고, 이전 세션과의 성장 비교 기능이 포함되어야 합니다.

[리포트 지표 정의(MVP)]
- 시선 지표(정량): 화면 응시율(%), 시선 분산 횟수(회), 시선 분산 총 지속 시간(초), 최장 분산 지속 시간(초)
- 세션 요약: 총 질문 수(개), 총 답변 시간(초) (타이머 기반)
- Q&A 요약: 질문별 사용자 답변 텍스트(사용자 입력 기반) 및 AI 요약(핵심 1~2문장)
- 개선 포인트 우선순위: 개선 항목 TOP3(각 항목에 근거 지표 1개 이상 연결)
- 성장 비교: 최근 5회 내에서 화면 응시율/분산 횟수/분산 지속 시간의 변화 추이를 그래프로 제공

[지표 계산/표시 규칙]
- 응시율 = (응시로 판정된 프레임 수 / 전체 측정 프레임 수) × 100
- 분산 횟수 = '분산 상태'가 연속 1초 이상 유지될 때 1회로 카운트(짧은 흔들림은 제외)
- 분산 지속 시간 = 분산 상태로 판정된 구간의 합(초)
- 지표 신뢰도 안내: 얼굴 미검출/조도 부족 등으로 측정 품질이 낮으면 '측정 불가/참고용'으로 표시

[비범위(v2)]
- 음성 STT 기반 발화 지표(말 속도/필러워드/침묵)
- 표정/감정/제스처 등 비언어 확장 지표

**수용 기준:**
```
1. ○ 면접 세션 종료 즉시 시선 처리 분석 결과와 답변 내용을 통합한 종합 리포트가 생성되어야 한다.
2. ○ 리포트에는 사용자가 개선해야 할 비언어적/언어적 요소(현재는 시선 처리 및 답변 내용)의 우선순위가 명확하게 제시되어야 한다.
3. ○ 사용자는 이전 면접 세션의 리포트와 현재 세션의 리포트를 비교하여 자신의 성장 추이를 확인할 수 있어야 한다.
4. ○ 리포트는 직관적인 시각 자료(그래프, 차트 등)를 포함하여 이해하기 쉽게 구성되어야 한다.
5. ○ 리포트에 화면 응시율(%), 시선 분산 횟수(회), 시선 분산 총 지속 시간(초), 최장 분산 지속 시간(초)이 포함되어야 한다.
6. ○ 분산 횟수는 '분산 상태'가 연속 1초 이상 유지될 때 1회로 카운트되어야 한다.
7. ○ 최근 5회 내 시선 지표(응시율/분산 횟수/분산 지속 시간)의 변화 추이가 리포트에 표시되어야 한다.
8. ○ 얼굴 미검출/조도 부족 등으로 측정 품질이 낮은 경우 리포트에 '측정 불가/참고용' 안내가 표시되어야 한다.
```


### 3.1 세션 피드백 리포트 생성/조회
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)

세션 종료 후 시선 지표와 Q&A를 종합한 리포트를 생성하고 조회합니다.


#### 3.1.1 리포트 구성(시선+Q&A)

리포트는 아래 구성으로 제공합니다(MVP).
- 시선 지표 요약: 화면 응시율(%), 시선 분산 횟수(회), 시선 분산 총 지속 시간(초), 최장 분산 지속 시간(초)
- 세션 요약: 총 질문 수(개), 총 답변 시간(초) (타이머 기반)
- Q&A 요약: 질문별 사용자 답변 텍스트(사용자 입력 기반) + AI 요약(핵심 1~2문장)
- 성장 비교: 최근 5회 내 시선 지표(응시율/분산 횟수/분산 지속 시간) 변화 추이를 그래프로 제공
- 지표 신뢰도 안내: 얼굴 미검출/조도 부족 등으로 측정 품질이 낮으면 '측정 불가/참고용'으로 표시

지표 계산/카운트 규칙
- 응시율 = (응시로 판정된 프레임 수 / 전체 측정 프레임 수) × 100
- 분산 횟수 = '분산 상태'가 연속 1초 이상 유지될 때 1회로 카운트(짧은 흔들림은 제외)
- 분산 지속 시간 = 분산 상태로 판정된 구간의 합(초)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)


#### 3.1.2 개선 포인트 우선순위

리포트에 개선 포인트 우선순위(TOP3)를 제공합니다.
- 각 개선 항목은 근거 지표 1개 이상과 연결합니다(예: 시선 분산 총 지속 시간↑ → '응시 유지 연습').
- 근거 지표는 최소 1개, 가능하면 2개(시선+세션요약)까지 연결합니다.
- 지표가 '측정 불가/참고용'인 경우, 해당 지표를 근거로 한 강한 결론을 피하고 '환경 개선 후 재측정' 안내를 포함합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)


#### 3.1.3 AI 피드백 생성 요청

세션 로그(질문, 사용자 입력 답변, 시선 지표, 세션 요약)를 바탕으로 AI에 피드백 생성을 요청합니다.
요청에 포함할 핵심 데이터(MVP)
- 질문 리스트(순서 포함)
- 질문별 사용자 답변 텍스트(사용자 입력)
- 질문별 AI 요약(핵심 1~2문장)
- 시선 지표: 화면 응시율(%), 시선 분산 횟수(회), 시선 분산 총 지속 시간(초), 최장 분산 지속 시간(초)
- 성장 비교용 히스토리: 최근 5회 내 시선 지표(응시율/분산 횟수/분산 지속 시간)
- 지표 신뢰도 상태: 정상/참고용/측정 불가(사유: 얼굴 미검출, 조도 부족 등)

요청/응답 규칙(요약)
- 개인정보가 포함될 수 있는 원본 카메라 영상/오디오는 요청에 포함하지 않습니다(기본값).
- 실패 시에는 최소 리포트(시선 지표만)로 대체할 수 있도록 응답을 분리해 저장합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)


#### 3.1.4 리포트 조회/공유(1차: 조회만)

MVP에서는 리포트를 앱 내에서 조회하는 기능을 우선 제공하고, 공유는 v2로 둡니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)

## 4. 면접 연습 기록 관리
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전

사용자가 진행한 모든 면접 연습 세션의 기록과 분석 리포트를 자동으로 저장하고, 사용자가 언제든지 과거 기록을 조회하고 재열람할 수 있도록 지원해야 합니다.

**수용 기준:**
```
1. ○ 사용자가 진행한 모든 모의 면접 세션의 기록(날짜, 시간, 면접 유형, 결과 리포트 링크 등)이 앱 내에 자동으로 저장되어야 한다.
2. ○ 사용자는 저장된 면접 기록 목록을 시간순으로 조회할 수 있어야 한다.
3. ○ 사용자는 목록에서 특정 세션을 선택하여 해당 세션의 상세 피드백 리포트를 다시 열람할 수 있어야 한다.
4. ○ 저장된 기록은 기기 로컬에 안전하게 저장되어야 한다.
```


### 4.1 연습 기록 히스토리(로컬)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)

세션 기록과 리포트를 기기 로컬에 저장하고 목록/상세로 재열람합니다.


#### 4.1.1 히스토리 목록(시간순)

세션 기록을 시간순으로 목록 조회합니다(면접 유형, 날짜/시간, 요약 점수 등).
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)


#### 4.1.2 히스토리 상세(리포트 재열람)

목록에서 세션을 선택하면 해당 세션의 리포트를 다시 열람할 수 있습니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)


#### 4.1.3 로컬 저장/삭제

세션 기록과 리포트를 기기 로컬에 저장하고, 사용자가 삭제할 수 있습니다.
- 저장 정책(MVP): 기본값은 무기한 보관(사용자가 직접 삭제하기 전까지). 용량/성능 문제 발생 시 앱 내에서 저장 용량 안내 및 삭제 유도를 제공합니다.
- 삭제 정책: 세션 단위 삭제(단일 세션 삭제)와 전체 삭제(모든 기록 삭제)를 제공합니다. 삭제 시 되돌릴 수 없음을 안내합니다.
- 삭제 범위: 세션 메타데이터(날짜/유형), 입력 텍스트(자기소개서/경험 요약), 요약 지표(시선), 리포트(Q&A 요약/피드백 텍스트)를 삭제합니다.
- 미저장/미삭제(기본값): 원본 카메라 영상/오디오는 저장하지 않으며, 따라서 삭제 대상도 아닙니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)


#### 4.1.4 개인정보 저장 범위 안내

로컬에 저장되는 항목과 저장하지 않는 항목, 그리고 사용자의 삭제 권한을 명확히 안내합니다.
- 저장(로컬): 세션 메타데이터(날짜/유형), 입력 텍스트(자기소개서/경험 요약), 요약 지표(시선), 리포트(Q&A 요약/피드백 텍스트)
- 미저장: 원본 카메라 영상/오디오(기본값)
- 삭제: 사용자는 세션 단위 삭제 및 전체 삭제를 할 수 있으며, 삭제 후에는 복구할 수 없음을 안내합니다.
- 유의: 로컬 저장은 동일 기기 내에서만 유지되며, 앱 삭제/기기 변경 시 기록이 사라질 수 있음을 안내합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)

## 5. 면접 세션 생성/설정(입력 정보)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

**수용 기준:**
```
1. ○ 사용자는 직무면접/인성면접/대학입시 면접 중 하나를 선택할 수 있어야 한다.
2. ○ 사용자는 직종(또는 전공/학과), 회사(또는 학교/학과) 정보를 입력할 수 있어야 한다.

5. ○ 입력 정보는 동일 기기에서 다음 세션 생성 시 재사용할 수 있도록 저장할 수 있어야 한다.
```

## 6. 기본 앱 흐름/권한/오류 처리
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

MVP 동작을 위해 카메라 권한, 네트워크 오류, AI 요청 실패 등 기본 동작/예외 처리를 제공해야 합니다.

**수용 기준:**
```
1. ○ 카메라 권한이 없을 때 권한 요청 및 권한 필요 안내를 제공해야 한다.
2. ○ 네트워크 오류 시 재시도 및 원인 안내를 제공해야 한다.
3. ○ AI 질문 생성/피드백 요청 실패 시 대체 안내 및 재시도 동작을 제공해야 한다.
4. ○ 세션 중 앱이 백그라운드로 가거나 화면이 꺼질 때의 동작(일시정지/종료)을 정의하고 안내해야 한다.
5. ○ 개인정보(카메라 영상, 텍스트 입력)를 포함할 수 있는 데이터의 저장 범위를 사용자에게 안내해야 한다.
```


### 6.1 카메라 권한 플로우
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Mobile (Android), Mobile (iOS)

#### 6.1.1 권한 상태별 분기(미요청/거부/영구거부)
