## 상위 3개 리스크 상세

### R-01: AI 코드 이해 부족
**상황**: Claude Code가 생성한 코드를 발표 때 설명하지 못함  
**대응**:
- AGENTS.md에 레이어 규칙 명세 → AI가 일관된 패턴으로 생성
- 커밋 전 직접 코드 리뷰 + `flutter analyze` 확인
- ADR에 모든 기술 선택 이유 기록 → 발표 때 ADR 기반 설명

### R-02: OpenAI gpt-4o-mini 실패
**상황**: 면접 중 API 타임아웃 또는 네트워크 오류  
**대응**:
- `_NetworkErrorInterceptor`로 오류 유형별 한국어 메시지
- `_fallbackImprovements()`로 시선 지표만으로 최소 리포트 생성
- `isAiFeedbackAvailable = false` 배너 표시

### R-03: 카메라 권한 거부
**상황**: 사용자가 카메라 권한을 거부하거나 영구 거부  
**대응**:
- `CameraPermissionScreen`에서 명확한 사용 목적 안내
- 영구 거부 시 `openAppSettings()` 로 설정 이동
- 권한 없이도 텍스트 답변 세션은 진행 가능하도록 설계
