# 요구사항 명세 (MoSCoW)

## 핵심 사용자 시나리오

### 시나리오 1 — 첫 면접 연습
> 취업 준비생 A는 백엔드 직무 면접을 앞두고 앱을 실행한다.  
> 자기소개서를 붙여넣고 회사명을 입력하자 AI가 맞춤 질문 3개를 생성한다.  
> 면접 중 카메라가 시선을 추적하고, 종료 후 응시율 72%와 개선 포인트를 확인한다.

### 시나리오 2 — 반복 연습 & 성장 확인
> A는 일주일 동안 5회 연습했다.  
> 히스토리 화면에서 응시율 추이 그래프(58%→72%)를 보며 성장을 확인한다.

### 시나리오 3 — 오프라인 / 오류 상황
> 지하철에서 앱을 켰을 때 오프라인 안내가 표시되고,  
> AI 질문 생성에 실패해도 시선 지표만으로 최소 리포트가 생성된다.

---

## Must (반드시 있어야 함)

| ID | 요구사항 | 검증 방법 |
|---|---|---|
| M-01 | 면접 유형(직무/인성/대입) 선택 | UI 선택 동작 확인 |
| M-02 | 직종·회사·자기소개서 입력 및 재사용 | Hive 저장·로드 확인 |
| M-03 | OpenAI gpt-4o-mini로 질문 3개 생성 | 실기기 API 호출 확인 |
| M-04 | 카메라로 실시간 시선 분석 | ML Kit 프레임 처리 확인 |
| M-05 | 시선 분산 1초 이상 연속 → 1회 카운트 | 단위 테스트 통과 |
| M-06 | 세션 종료 후 리포트 생성 (AI + 시선 지표) | 리포트 화면 렌더링 확인 |
| M-07 | AI 실패 시 fallback 리포트 생성 | 단위 테스트 통과 |
| M-08 | 히스토리 목록 저장 및 상세 재열람 | Hive 저장·로드 확인 |
| M-09 | 단일·전체 기록 삭제 | 삭제 후 목록 갱신 확인 |
| M-10 | 카메라 권한 요청·거부·영구거부 처리 | 분기 동작 확인 |

## Should (있으면 좋음)

| ID | 요구사항 |
|---|---|
| S-01 | 꼬리 질문 생성 (답변 기반) |
| S-02 | 최근 5회 응시율 추이 그래프 |
| S-03 | 세션 입력 확인 화면 + 개인정보 안내 |
| S-04 | 질문 로딩 중 스켈레톤 UI |
| S-05 | 오프라인 감지 및 안내 메시지 |
| S-06 | 앱 백그라운드 전환 시 타이머 정지 |

## Could (여유 있으면)

| ID | 요구사항 |
|---|---|
| C-01 | Firebase App Distribution 배포 |
| C-02 | 시선 분석 상세 통계 (구간별 분산 타임라인) |
| C-03 | 답변 음성 입력 지원 |

## Won't (이번 MVP 제외)

- 서버 저장 / 클라우드 동기화
- 소셜 기능 (공유, 랭킹, 친구)
- 유료 결제 / 구독 플랜
- 웹 버전

## 비기능 요구사항

| 항목 | 기준 |
|---|---|
| API 타임아웃 | 30초 초과 시 오류 메시지 |
